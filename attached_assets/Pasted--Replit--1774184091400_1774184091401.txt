너는 시니어 풀스택 개발자다. 아래 요구사항을 만족하는 완성도 높은 웹앱을 Replit에서 바로 실행 가능하게 만들어라.

[프로젝트 목표]
청각장애인을 위한 “실시간 자막 + 핵심 요약 + 중요 문장 자동 표시” 웹앱을 만든다.
이 앱은 공모전 시연 영상 제출용이며, 심사에서 실용성, 신규성, 아이디어 구체성, 실현 가능성, 완성도가 잘 드러나야 한다.
기존 자동 자막 서비스와 차별화되도록 “중요도 자동 표시”와 “짧은 요약”을 핵심 기능으로 넣는다.

[중요 조건]
- 반드시 한국어 UI로 만든다.
- 마이크 입력을 실제로 받는다.
- 외부 유료 API나 클라우드 API는 사용하지 않는다.
- 인터넷 없이도 핵심 기능이 동작해야 한다.
- Replit에서 바로 실행되어야 한다.
- 앱은 브라우저에서 열리는 웹앱이어야 하며, 단일 포트로 동작해야 한다.
- 시연 영상에서 기능 변화가 즉시 보이도록 화면 구성을 직관적으로 만든다.
- 실패하지 않도록 fallback 로직을 넣는다.

[권장 기술 스택]
- Python 3.11
- FastAPI 또는 Flask 중 하나를 선택하되, Replit에서 안정적으로 실행되는 쪽으로 선택
- 프론트엔드: HTML / CSS / Vanilla JavaScript
- 오디오 입력: 브라우저의 MediaRecorder 또는 getUserMedia로 마이크를 캡처
- 음성 인식: 로컬 오프라인 Whisper 계열 또는 Replit 환경에서 실행 가능한 대체 방식
- 요약 및 중요도 판정: 로컬 규칙 기반 + 가능하면 로컬 LLM 연동 구조
- 실시간 업데이트: fetch polling 또는 WebSocket 중 구현이 더 단순하고 안정적인 방식

[핵심 기능]
1. 실시간 자막
- 사용자가 마이크로 말하면 화면에 자막이 거의 실시간으로 쌓인다.
- 완전한 스트리밍이 어렵다면 2~5초 단위 청크 처리로 구현한다.
- 자막은 최근 발화가 아래로 누적되며 읽기 쉽게 보여야 한다.

2. 중요도 자동 표시
- AI 또는 규칙 기반으로 문장 중요도를 판단한다.
- 중요 문장은 시각적으로 강하게 강조한다.
- 예: 날짜, 시간, 장소, 시험, 마감, 제출, 주의, 위험, 중요, 반드시, 변경, 내일, 오늘 같은 키워드가 포함되면 강조 점수를 높인다.
- 핵심 문장은 색상 강조, 배지, 하이라이트 중 최소 1개 이상으로 표시한다.
- 긴 문장 속 핵심만 빠르게 파악할 수 있도록 한다.

3. 실시간 짧은 요약
- 최근 발화 내용을 1~3문장 정도로 짧게 요약해서 보여준다.
- 사용자가 지금 어떤 이야기를 듣고 있는지 구조를 빠르게 파악할 수 있어야 한다.
- 요약은 너무 길지 않게 유지한다.

4. 시연용 예시 버튼
- 아래와 같은 예시 시나리오 버튼을 넣는다.
  - 수업 공지
  - 회의 안내
  - 시험 일정 안내
- 버튼을 누르면 샘플 문장이 들어가서 시연 영상에서 기능이 빠르게 드러나야 한다.

5. 화면 구성
- 상단: 앱 제목과 짧은 설명
- 상단 또는 좌측: 마이크 시작 / 정지 버튼, 상태 표시
- 중앙: 실시간 자막 영역
- 우측 또는 하단: 핵심 요약 영역
- 자막 영역 안에서 중요 문장은 자동 강조
- 전체 UI는 심플하고 가독성이 좋아야 한다
- 공모전 발표용이므로 한눈에 기능이 보이게 배치한다

[오프라인 동작]
- 외부 네트워크가 없어도 핵심 기능이 동작해야 한다.
- 외부 API 호출이 없도록 한다.
- 가능한 범위에서 로컬 처리만 사용한다.
- 로컬 모델이 없을 때를 대비한 규칙 기반 fallback을 반드시 넣는다.

[구현 요구]
- Replit에서 실행 가능한 코드로 작성한다.
- 파일 구조를 정리해서 제공한다.
- 최소한 다음 파일을 포함한다:
  - main.py 또는 app.py
  - templates/index.html 또는 equivalent
  - static/style.css
  - static/app.js
  - requirements.txt
  - README.md
- 실행 방법이 README에 명확히 적혀 있어야 한다.
- 에러가 나도 앱이 완전히 죽지 않도록 예외처리를 넣는다.
- 코드에는 주석을 충분히 달아라.
- 지나치게 복잡한 기업용 기능은 빼고, 공모전 핵심 기능만 완성도 있게 구현한다.

[심사 기준 반영]
- 실용성: 청각장애인의 실제 사용 장면을 고려한다.
- 신규성: 단순 자막이 아니라 중요도 표시와 짧은 요약을 결합한다.
- 아이디어 구체성: 자막, 요약, 강조 방식이 명확해야 한다.
- 실현 가능성: Replit에서 바로 돌아갈 정도로 구현 가능해야 한다.
- 완성도: 시연 영상에서 “말하면 자막이 뜨고, 핵심이 강조되고, 요약이 갱신되는 흐름”이 명확해야 한다.

[출력 형식]
1. 먼저 전체 설계 개요를 간단히 설명한다.
2. 그다음 파일별로 코드를 모두 제공한다.
3. 마지막에 실행 방법을 단계별로 정리한다.
4. 마지막으로 시연 영상에서 꼭 보여줄 장면 3가지를 정리한다.

[추가 지시]
- 제목은 공모전 제출용으로 어울리게 정한다.
- 너무 기술적으로만 보이지 않게, 사용자 친화적인 문구를 사용한다.
- 자막, 요약, 중요 표시가 모두 실제로 확인 가능해야 한다.
- 기본 테마는 깔끔한 밝은 테마로 한다.
- 한국어 문장 처리에 맞게 설계한다.
- 가능하면 로딩 상태, 오류 상태, 마이크 권한 상태도 보여준다.

이 요구사항을 만족하는 완성된 프로젝트를 생성해라.
